# Specification: Showtime-Aware Recommendation Filter

## Overview
Modify the movie vector recommendation service to only suggest movies that have at least one showtime scheduled for the current day. This ensures that the AI assistant provides relevant recommendations that users can act upon immediately.

## Functional Requirements
1. **Integration with Backend:** The `movie_vector_service.py` must fetch the list of currently showing movies from the Node.js backend using the existing `get nowshowing movies` API endpoint.
2. **Pre-filtering Search:** Recommendations generated by ChromaDB must be filtered to only include movies from the "now showing" list. This filtering should happen *before* retrieving results to ensure all top results are currently showing.
3. **Consistency with Backend:** The filtering must respect the backend's logic for "now showing" (typically movies with showtimes starting after the current time today).
4. **Error Handling:** If no movies are currently showing or the backend API call fails, the service should return an empty list of recommendations.

## Non-Functional Requirements
1. **Performance:** Efficient retrieval of the "now showing" movie list, ideally with a minimal impact on recommendation latency.
2. **Reliability:** Graceful handling of backend connectivity issues.

## Acceptance Criteria
- [ ] Recommendations only include movies with showtimes available on the current day.
- [ ] The filtering logic is consistent with the backend's "now showing" movies list.
- [ ] If no movies have showtimes today, an empty result or appropriate message is returned.
- [ ] Existing vector similarity rankings are maintained within the filtered subset.

## Out of Scope
- Modifying the backend's "now showing" logic.
- Adding recommendations for upcoming movies (beyond today).
